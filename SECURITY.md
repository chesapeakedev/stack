# Security

## Reporting vulnerabilities

If you discover a security vulnerability in this project, please report it
responsibly.

**Do not** open a public GitHub issue for security-sensitive bugs.

- **Email**: Report to the maintainers via the contact method listed in the
  project repository (e.g. GitHub profile or `package.json` author field).
- **Describe** the issue and steps to reproduce, if possible.
- **Allow** a reasonable time for a fix before any public disclosure.

We will acknowledge your report and work on a fix. We appreciate responsible
disclosure.

## Supported versions

Security updates are applied to the current major version. Upgrade to the latest
release to receive fixes.
